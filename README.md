# CIS Controls Implementation & Access Governance Framework  
### SLA Security Requirements for IoT Health Platform Partnership  

## Project Overview

This project establishes SLA-aligned security requirements using CIS Controls to reduce cross-organizational risk exposure in a health IoT partnership environment.

---

## Skills Demonstrated

- CIS Controls interpretation and implementation
- Role-Based Access Control (RBAC) design
- Multi-Factor Authentication (MFA) policy development
- Account lifecycle automation planning
- Access governance auditing strategy
- Security awareness program design
- Risk mitigation and defense-in-depth modeling

---

## Key Controls Addressed

- CIS Control 5: Account Management  
- CIS Control 6: Access Control Management  
- CIS Control 14: Security Awareness & Skills Training  

---

## Outcome

Designed enforceable SLA security requirements integrating RBAC, MFA, automated account governance, and structured social engineering defense programs to reduce credential compromise and insider risk exposure.
